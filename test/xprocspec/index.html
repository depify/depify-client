xprocspec - XProc testing tool

This is a tool for testing XProc scripts. Its grammar is based on XSpec and is implemented in XProc.

xprocspec is itself tested using xprocspec. The latest results are available as a xprocspec HTML report. xprocspec is at this point mostly feature-complete.

Command line

From the command line, provide the xprocspec test on the "source" input port, and point the "result", "html" and "junit" report ports to a location of your choice. Assuming you have calabash 1.0.15-95 installed under ~/calabash-1.0.15-95 (calabash 1.0.10 or newer is recommended), cd to the "xprocspec" directory in the xprocspec repository and run the following command to run the xprocspec integration tests:

java -jar ~/calabash-1.0.15-95/calabash.jar -i source=src/test/xprocspec/test.xprocspec -o result=target/result.xml -o html=target/html.html -o junit=target/junit.xml src/main/resources/content/xml/xproc/xprocspec.xpl

The xprocspec reports, including HTML and JUnit versions, will be stored in "xprocspec/target/".

NOTE: In Windows, you must also provide a file URI to the "temp-dir" option. It defaults to "file:/tmp/" which works in Linux and OSX.

Maven

If you use the Maven plugin @daisy-consortium/xproc-maven-plugin, you can do automated testing of your XProc projects!

The simplest way to start an XProc project with xprocspec testing is to simply run the following command:

mvn archetype:generate -DarchetypeRepository=https://oss.sonatype.org/content/repositories/snapshots/ -DarchetypeGroupId=org.daisy -DarchetypeArtifactId=xprocspec-archetype-basic -DarchetypeVersion=1.0.0-SNAPSHOT -DgroupId=org.example -DartifactId=my-xproc-project

This will create the basic folder structure of your project, and includes a Hello world XProc pipeline as well as a xprocspec test for it. To test the project, run mvn test.
